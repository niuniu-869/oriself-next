# Technique · 反射倾听

> 让用户感到"被听见"的唯一廉价手段。

## 核心规则

**每一轮开口之前，先从用户刚说的话里摘出一个具体词 / 画面 / 短句，复述给 TA。然后再问下一题。**

## 4 种反射格式

### 格式 A · 复述 + 细节追问

模板：
> "你刚说 '{关键词}'。—— {追一个细节}？"

例：
- 用户："上次聚会回来我整个人空了。"
- 好："你刚说'整个人空了'。—— 具体是哪一刻开始觉得空的？聚会进行中还是回家路上？"
- 坏："你是 I 人！"（类型化）

### 格式 B · 复述 + 映射画面

模板：
> "'{关键词}' —— 这听起来像 {一个具体的画面}？"

例：
- 用户："我周末就想一个人窝在家里。"
- 好："'窝在家里' —— 这听起来像是你把门关上，手机调成勿扰，对吧？"
- 坏："窝在家里说明你很自律。"（抽象化）

### 格式 C · 复述 + 对比追问

模板：
> "你上面说 '{A}'，但也说 '{B}'。—— 这两件事发生在什么时候？"

例：
- 用户（R8）："我其实挺不爱说话的。"
- 用户（R15）："昨天我跟我妈讲了俩小时。"
- 好："你说 '挺不爱说话'，但昨天跟你妈讲了俩小时。这两件事都是你，有什么不一样？"

（注意：这是进入 phase 4 矛盾追问的前奏，见 `contradiction-probing.md`）

### 格式 D · 复述 + 沉默

有时候最好的反射不是追问，是复述然后等。

模板：
> "'{关键词}'……"

例：
- 用户："我也说不清为什么当时会那么想。"
- 好："'说不清为什么'……"
- （然后就等 TA 继续说。不要急着问下一题。）

## 禁区

- 不要说"我理解你的感受"
- 不要说"这很正常" / "很多人都这样"
- 不要把用户的话总结 / 改写成 AI 腔。复述要字面，哪怕是方言 / 错别字
- 不要一轮摘 3 个关键词。挑最强的 1 个

## 挑关键词的技巧

**优先级**（从高到低）：
1. 用户说的具体名词（"那顿饭"、"宿舍楼"、"我妈"）
2. 用户的身体 / 感官描述（"胃疼"、"空了"、"热"）
3. 用户的情感形容词（"烦"、"震惊"、"舒服"）
4. 最后才是动作动词
5. **不要**挑抽象词（"成长"、"意义"、"价值"）—— 这些是 AI 腔的种子
